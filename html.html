HTML5 Fundamentals

What is HTML5?

HTML5 is a more recent version of HyperText Markup Language that has more semantice as well as multimedia features and more accessibility. It is the basis of all modern web sites, and does not conflict with CSS or JavaScript.

Semantic HTML elements for better structure
Built-in accessibility features
Native multimedia support
Mobile-first responsive design

Semantic HTML5 Elements

Semantic elements should be meaningful and should create well structured, accessible websites that search engines adore.

<header>

Includes the introduction, navigation and branding content.

Example: Site logo, navigation menu, search bar

<nav>

Defines a navigation space that has links to other pages or areas on the same page.

Example: Main menu, breadcrumbs, pagination

<main>

The body of the document, but not headers, footers and navigation.

Example: Article content, form sections, key information

<article>

Is a stand-alone composition which may be distributed separately.

Example: Blog posts, news articles, forum posts

<section>

Identifies a part of a document which is usually accompained by a heading.

Example: Chapters, tabbed content, grouped content

<footer>

Site information like contact links, copyright details and other related references are in the element of the footer.

Example: Contact info, social links, legal notices

Modern Forms & Inputs

HTML5 introduces powerful new form elements and input types for better user experience and validation.

Enhanced Input Types

// Modern HTML5 input types
<input type="email" placeholder="Enter email" required>
<input type="tel" pattern="[0-9]{10}">
<input type="date" min="2020-01-01">
<input type="color" value="#ff6b6b">
<input type="range" min="0" max="100">
<input type="search" placeholder="Search...">

Form Validation & Accessibility

// Accessible form with validation
<form novalidate>
<label for="username">Username:</label>
<input id="username" type="text" required aria-describedby="username-help">
<div id="username-help" class="help-text">Must be at least 3 characters</div>
<button type="submit">Submit</button>
</form>

HTML5 Best Practices

Do's
  • Use semantic HTML5 elements for better structure
  • Include proper alt attributes for all images
  • Use meaningful heading hierarchy (h1-h6)
  • Check your HTML code often
  • Have a clean and well intended code
  • Use descriptive link text
  • Implement proper form labels and validation
  • Ensure keyboard navigation support
Don'ts
  • Don't use deprecated HTML4 tags
  • Avoid inline styles and JavaScript
  • Don't skip closing tags
  • Don't use tables to layouts
  • Accessibility features are to be remembered
  • Use semantic elements where you can
  • Never employ old character encoding
  • Avoid excessive nesting of elements
